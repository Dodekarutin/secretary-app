# AGENTS.md

## プロジェクト概要

- このプロジェクトは雑に投げたタスク、やりたいことを細分化し、それらをモダンな UI で表示することを目的としており、主要な技術スタックは React, TypeScript, Node.js です。
- ディレクトリ構成：
  - `/src` ─ メインのソースコード
  - `/tests` ─ テストコード
  - `/public` ─ 静的資産
  - `/scripts` ─ ビルドやデプロイ用スクリプト
- このプロジェクトはテスト駆動開発で進めること

## 開発環境

- 開発には Node.js 18 を使ってください。
- 依存関係のインストール：`npm install`
- ローカル開発サーバー起動：`npm run dev`
- 使用しているパッケージマネージャーは pnpm。

## ビルド / テスト

- ビルドコマンド：`npm run build`
- テスト実行：`npm test`
- カバレッジ取得：`npm test -- --coverage`
- Lint チェック：`npm run lint`
- 型チェック：`npm run type-check`

## コードスタイル / 規約

- クォーテーションは（"）を使う
- セミコロンは使わない
- インデントは 2 スペース
- ファイル名・ディレクトリ名は camelCase ではなく kebab-case
- 非同期関数は `async/await` を使う

## PR / コミットメッセージ規則

- PR のタイトルは `[コンポーネント名] 概要` の形式を使う
- コミットは小さな変更ごとに分ける。テストが通る状態であること
- マージ前に `lint` と `test` がパスすること

## セキュリティ注意事項

- 環境変数に秘密情報を含めないでください。`.env` の内容は `.gitignore` に追加する。
- 外部 API キーなどは Vault または安全なストレージを使うこと
- 入力検証を怠らないこと。特にユーザー入力や外部データ取得部分。

## デプロイ / 運用

- デプロイ先環境：ステージング → 本番
- CI/CD を使って自動的にビルド＆テスト → デプロイを行う

## その他

- ドキュメント、API 仕様書、ER 図などは随時更新すること
